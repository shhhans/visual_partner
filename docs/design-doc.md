## 2. 运营成本控制

### 想到的技巧

| # | 技巧 | 采用？ | 说明 |
|---|---|---|---|
| C1 | 视觉按需化：VL 调用由 LLM 工具触发，闲聊零视觉成本 | ✅ 计划采用 | 对比常驻视频流方案，成本差一个数量级以上 |
| C2 | 帧端侧预处理：缩图至 ≤768px + JPEG 压缩，再上传 | ✅ 计划采用 | VL 按图块计 token |
| C3 | 模型分层：qwen-plus 做大脑而非 max；VL 可降级 vl-plus | ✅ 已采用 | |
| C4 | system prompt 约束短回答、口语化 | ✅ 已采用 | 输出 token 是大头（且 TTS 按字计费） |
| C5 | 滚动历史窗口，图片不进历史，VL 结果以文本入史 | ✅ 计划采用 | |
| C6 | barge-in 打断即停止 LLM 生成与 TTS 合成 | ✅ 已采用 | 被打断的内容不再付费生成 |
| C7 | ASR 静默期不送音频（端侧能量门限） | ⬜ 评估中 | paraformer 按音频时长计费 |
| C8 | VL 结果短期缓存：同帧/近帧 N 秒内复用描述 | ⬜ 评估中 | |
| C9 | Context cache（DashScope 隐式缓存命中 system prompt） | ⬜ 评估中 | 长 system prompt 时收益明显 |
| C10 | 端侧免费能力优先：浏览器做回声消除/降噪/缩图，不上云 | ✅ 已采用 | |
